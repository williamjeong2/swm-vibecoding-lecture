MCP vs CLI / 23

CLI가 더 싼 선택일 때가 있다

MCP는 편하지만 도구 설명이 컨텍스트를 먹습니다. 공식 CLI가 성숙하면 토큰과 기능 양쪽에서 유리합니다.

판단 규칙

반복 작업은 CLI 래퍼로, 대화형 탐색은 MCP로 시작합니다.

CLI wrapper

토큰

help, json, exit code만 넘기면 충분합니다.

기능

공식 CLI가 최신 옵션을 먼저 갖는 경우가 많습니다.

자동화

shell pipeline, CI, 재시도 스크립트에 강합니다.

주의

출력 파싱과 권한 경계는 직접 잡아야 합니다.

MCP tool

토큰

도구가 많아지면 설명 자체가 컨텍스트 비용이 됩니다.

기능

구조화된 호출은 좋지만, 옵션이 늦게 열릴 수 있습니다.

탐색

대화형 조회와 객체 단위 작업에 자연스럽다.

주의

기본으로 켜두는 도구 수는 작게 유지해야 합니다.
